# Sidet

Sidet，中文名“思得”，是一个面向 Obsidian 的轻量 AI 对话插件。它把对话浏览放在侧边，把当前聊天保持在正中，并让有价值的结果更容易回到你的 vault。

这个公开仓库只发布安装所需文件，不包含源代码。

## 为什么是 Sidet

很多笔记软件里的 AI 插件都偏重：配置多、切换多、操作绕，真正开始对话之前就已经有不少负担。

Sidet 走的是更轻的路径：

- 配置更简单
- 对话流程更符合直觉
- 在对话和笔记之间切换更顺
- 有价值的结果更容易继续回到你的 vault

## 你能得到什么

- 更简单的配置方式，不走厚重的配置流程
- 更快切换对话的侧边浏览
- 更符合直觉的主聊天视图，适合连续对话
- 默认隐藏保存、按需导出的会话记录
- 更方便把有价值的结果继续整理回你的 vault

## 主要体验

### 侧边浏览 + 主聊天区

- 在侧边浏览和切换对话，不轻易打断当前会话
- 当前聊天保持在主区域，更适合连续阅读和输入
- 整体布局保持简单，但切换效率更高

### 把主页当成你自己的入口页

- Sidet 在 Obsidian 里有自己的主页
- 你可以把常用命令固定到主页
- 常见操作不必反复翻菜单，进入就能用

### 更轻的上下文带入方式

Sidet 目前支持几种比较自然的上下文输入方式：

- 引用 Markdown 笔记
- 引用网页
- 选择图片
- 用 `@` 快速引用 Markdown 笔记

这些内容会作为上下文附件进入对话，而不是直接把一大段内容塞进输入框里，整体会更干净。

### 更容易整理到知识库

Sidet 不只是回答问题，也支持把有价值的结果继续整理回你的 vault。

- 可以把当前对话整理到知识库
- 也可以把已有笔记带入同一条知识整理流程
- 写文件前仍然是确认优先，先看预览，再决定是否写入

### 更清楚的知识整理结构

Sidet 当前的知识主线以 `Sidet/Wiki` 为中心，结构比较轻，但已经能支持后续维护：

- `knowledge-card`：适合高密度长期知识
- `concept-page`：适合更大的主题页
- `supporting-note`：适合补充性材料
- `Index.md`：作为顶层入口页

重点不是一次性堆内容，而是让知识后续更容易回看、补充和继续整理。

### 更顺手的网页、笔记和图片带入

- 网页可以引用进对话，并在发送前先预览
- Markdown 笔记可以作为干净的附件带入
- 图片可以作为上下文进入当前对话
- 有价值的结果可以导出，或通过确认式写回继续整理进 vault

## 安装

1. 从最新 Release 下载文件
2. 放入：

```text
.obsidian/plugins/sidet/
```

3. 确认目录中包含：
   - `main.js`
   - `manifest.json`
   - `styles.css`

## 可见命令与快捷入口

Sidet 已经支持一些适合固定到主页，或者作为常用快捷命令的入口，包括：

- 打开 Sidet
- 新建对话
- 导出当前对话
- 把当前笔记插入草稿
- 把当前选区插入草稿
- 把当前对话整理到知识库

这样做的目的，是把主路径保持清楚，而不是把整个插件做成一个很重的控制台。

## 公开仓范围

这个仓库只包含发布文件：

- `main.js`
- `manifest.json`
- `styles.css`
- `versions.json`
